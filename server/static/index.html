How It Works

You have 10,000 units to buy over 500 ticks. A synthetic market provides liquidity through an order book. Every time you buy, you push the price up. Other players' agents are buying simultaneously, competing for the same liquidity.

Your score is how much extra you paid vs the starting price, in basis points (1 bps = 0.01%). Lower is better. Negative means you beat the starting price.

Example: the starting price is $100. A score of 50 bps means your average fill was $100.50 — on 10,000 units that's $5,000 in extra cost. A score of -10 bps means you averaged $99.90, beating the market.

Mean = your average score across all tournaments. Best = your best single tournament. Each tournament runs 50 random seeds and averages.

Getting Started

  1. Clone: git clone https://github.com/Wesius/vibecoding-comps.git
  2. Install: uv sync
  3. Drop your .txt credentials file in the repo root, rename to .env
  4. Edit agent/agent.py
  5. Test: uv run python cli.py test
  6. Submit: uv run python cli.py submit

Order Types

Market orders fill immediately but you pay the spread and walk the book if it's thin.

Limit orders are cheaper if they fill, but unfilled orders are cancelled at end of tick.

Your agent sees the full order book, trade tape, and your own fill history every tick.
